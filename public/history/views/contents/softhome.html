학과 홈페이지를 가보면 알 수 있듯이 구글, 파이어폭스에 홈페이지가 원활히 보여주진 않는다.
여러 버그들이 존재하는 학과 홈페이지라 개선이 필요하게 되었다.
학과 홈페이지를 만들게 되었고, 디자인과 내부적인 속도 향상, 다양한 브라우저 환경을 보여주도록 하였다.

Development

백엔드(JSP, Servlet), 프론트 엔드(웹 화면)을 담당하였다.
웹 화면 설계는 Prototype.js와 jquery ajax를 이용하여 웹 화면 전환 없이 콘텐츠 부분만 전환하게 구성하였다.
예를 들면, URL 부분은 #를 이용하여 index.jsp#board?key=value
index.jsp#article?key=value 형식으로 각각의 URL의 형태에 따라 웹 컨텐츠 화면 부분이 달라지게 하는 것이다.
일반 URL과 다른 점은 #기준으로 URL을 자바스크립트가 식별한 다음, ajax 기술을 이용해 웹 페이지를 전환하게 해주는 것이다.
다만, Ajax를 사용하면 뒤로가기 하였을 경우, 이전 페이지 접근이 힘들다는 점을 보완하기 위해 hashchange.js를 이용하여 ajax를 이용하여도 뒤로가기가 되게 구현하였다.
서버는 서브 메뉴별 servlet을 두어 MVC 모델로 요청에 따라 다른 전송을 하였다.
